## Wireframe (1h30)

Now that you know which Design System to use, it's time to look back at your Core User Journey and create your wireframe based on it.
The goal is to create a lo-fi mockup for each screen of your product: you should not worry about colors, fonts, images or icons at this stage.

**⚠️ Important:** When designing each screen, keep in mind all the work you have done so far.
- What **single action** do you want your user to take on each page?
- What **context** do they need to take that action?

### Create your first screen

In your newly created Design project, start by creating a new `Page`, rename it properly (e.g. Prototype) and add a new `Frame` in it.

You can either add `80px` margins on both sides of the frame, or use a [Layout Grid](https://help.figma.com/hc/en-us/articles/360040450513-Create-layout-grids-with-grids-columns-and-rows).

When your first screen is created, you can simply duplicate it by `ctrl+D` or `Cmd+D` (D is for "duplicate"). This way, each screen will keep the same margins. Create as many screens on your Figma as you have screens in your Core User Journey.

**Be careful here** : it's very important that you create a Figma's `Frame` for each **Screen** of your app. If you're not sure of your setup, please call a teacher to validate it.

When all the screens are created, each member of your team can take care of one screen (you can also work by groups of 2).

**Figma pro tips:**
- `Cmd+D` / `ctrl+D` is the **Duplicate** shortcut - it is your best friend
- During prototyping, you can only connect a `Frame` to another `Frame`

### Using components

Besides offering a single source of truth for design, working with a Design System gives you access to an extensive library of components that you can simply drag and drop.

Some typical components you can look for:
- Button
- Input
- Card

Most components are customizable, and you will be able to edit the placeholder text, change its state, or add images for example.
