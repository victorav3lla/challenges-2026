## 核心用户旅程 (30分钟)

现在你有了清晰的产品推介，整个团队也有了共同的愿景，是时候识别并绘制你产品的**核心用户旅程**了。

### 什么是核心用户旅程？

核心用户旅程是用户使用你的应用程序来实现其目标时会采取的**主要行动**。这是你应用程序中最重要的任务，也是你今天将要**在全班面前展示**的内容。

如何识别你的核心用户旅程？
对于大多数现代 web 应用程序，核心旅程满足以下一个或两个条件：
- 它**满足目标用户的需求**
- 它产生**收入**

在 FigJam 的 **UX Research 模板**中，将你的核心用户旅程写在右上角的框中。

### 绘制用户旅程

在现代 web 应用程序中，用户可以通过不同的旅程完成多项任务。其中一些任务几乎所有产品都有（注册、登录、编辑个人资料等），而其他任务则是每个应用程序特有的。**注册/登录流程在此阶段不需要包含在你的原型中**，但将成为你最终产品的一部分。我们将使用 Ruby gem 来管理它们，所以你不需要在这次设计冲刺中花时间处理它们。

现在你已经识别了核心用户旅程，你可以通过识别以下内容来绘制它：
- 你的用户将浏览哪些**页面**（或屏幕）
- 你的用户需要在每个页面上采取什么**行动**
- 你的用户需要什么**上下文**来采取该行动

例如，以 Airbnb 为例：
- **我的用户在：** 主页
- **我希望我的用户：** 搜索住宿地点
- **为此，他们需要：** 指定目的地、旅行日期、旅行人数

![用户旅程](https://wagon-public-assets.s3.eu-west-3.amazonaws.com/j0xhy7gsg3dlu03hy71gqp6anrz2)

为了使你的核心用户旅程尽可能高效，请记住一些简单的 UX（用户体验）规则：
- 不超过**五个页面**（每多一个页面你就会失去用户！）
- 在核心旅程结束时，你的用户应该已经满足了他们的需求（例如，预订了住宿）
- 没有**多选择旅程** - 它是一条直线

**重要提示：** 在今天的练习中，你不需要构建登录屏幕。你可以假设你的用户已经登录，或者他们不需要登录就可以使用你的应用程序。

### 轮到你了！

回到 FigJam 上的 **UX Research 模板**，逐页绘制你的核心用户旅程。

完成后，确保**叫老师**来验证你的核心用户旅程。
